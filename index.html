XDL Bridge — formati multipli ↔ XDL

Questa pagina usa XDL come linguaggio intermedio unificato. Puoi caricare file Word (.docx), Excel (.xlsx, .xls, .ods), PowerPoint (.pptx), PDF, JSON, XML, database SQLite / SQL, messaggi clinici HL7/FHIR oppure un file XDL (.xdl o .txt). Il sistema genera e mostra l’XDL a video e poi ti permette di ricostruirlo nel formato di destinazione scelto. I campi dell’interfaccia compaiono solo quando servono davvero in base al formato selezionato.

1. Sorgente e conversione intermedia

Scegli il file di input. I campi specifici compaiono solo quando sono pertinenti al tipo di file selezionato. Per Excel puoi indicare foglio e intervallo; per gli altri formati vedi solo le opzioni realmente utili.
Nessun file sorgente selezionato.
Estrae soprattutto paragrafi, titoli e tabelle leggibili.
Estrae struttura slide, titoli, testi e note principali.
L’estrazione è principalmente testuale, pagina per pagina.
Conserva struttura e contenuto; gli oggetti profondi possono essere appiattiti.
Conserva nodi, attributi e testo in forma strutturata.
Legge tabelle e struttura di base; elementi avanzati possono essere semplificati.
Supporto base a HL7 v2 e FHIR in forma semplificata.
Se il sorgente è già XDL, puoi ricaricarlo, visualizzarlo e salvarlo subito.

Anteprima XDL

In attesa di un file sorgente oppure di un testo XDL incollato manualmente.

2. Destinazione e ricostruzione

Qui la sorgente deve essere un file XDL oppure il testo XDL già presente a sinistra. Scegli il formato di uscita desiderato. Le opzioni specifiche di Excel compaiono solo se il formato di destinazione è davvero Excel.

Riassunto del modello interno

Nessun modello XDL disponibile.

Log operativo

Nessuna esportazione eseguita.
Excel ↔ XDL Word → XDL → Word PowerPoint → XDL → PowerPoint JSON ↔ XDL XML ↔ XDL PDF ↔ XDL Database ↔ XDL Clinico ↔ XDL

Il programma privilegia una conversione trasparente: quando non può conservare perfettamente layout, formule vive, animazioni o dettagli grafici, li semplifica e li dichiara nel log.
